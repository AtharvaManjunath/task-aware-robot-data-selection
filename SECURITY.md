# Security Policy

This repository is research software and does not currently provide a formal security support window.

Please report security-sensitive issues privately to the repository owner or maintainer once ownership/contact details are added. Do not open public issues for vulnerabilities involving unsafe file handling, dependency-chain concerns, or private dataset leakage.

## Data Privacy

Do not commit private robot datasets, human demonstration videos, credentials, or experiment logs containing sensitive metadata. Use ignored local directories such as `data/`, `datasets/`, and `outputs/`.
